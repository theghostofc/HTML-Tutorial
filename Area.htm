AREA

This is to be used in the MAP Tag. This defines a Client Side Image Map Definition. It
indicates an Area on the Image where the users can choose to link to another document or location. This is not a Container Tag and thus need not be closed.

Attributes

  1. SHAPE :
    1. RECT : To create Rectangular Links.
    2. DEFAULT : This includes the Area on the Image on which no Link has been created.
    3. CIRCLE : To create Circular Links.
    4. POLY : To create Polygon Figures as Links.
  2. COORDS : Specifies the Coordinates of the Shape. The DEFAULT and RECT take two points as X1,Y1,X2,Y2. The CIRCLE takes three points as X,Y,RADIUS. The POLY takes many points as X1,Y1,X2,Y2,X3,Y3,.....
  3. HREF : Specifies the Path of the Document or Location to be loaded when this link is Activated.
  4. TARGET : Identifies in which Named Frame to load the document.
    Please refer to the FRAME Tag for more information about TARGET Attribute

Syntax

<MAP>
<AREA [SHAPE="TYPE"] [COORDS="X1,Y1,.."] [HREF="URL"] [TARGET="NAME"]>
</MAP>
To use this tag please refer to the IMG Tag.

Example

<MAP NAME="MP1">
<AREA SHAPE="RECT" COORDS="1,1,100,100" HREF="HTTP://WWW.YAHOO.COM"> </MAP>
