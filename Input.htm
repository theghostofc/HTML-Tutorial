INPUT

This is to be used in the FORM Tag. This Tag identifies several Input Methods for Forms.
This is not a Container Tag and thus need not be closed.

Attributes

  1. ACCEPT : Specifies a list of Acceptable MIME Types for Submitting Files .
  2. CHECKED : Used with the types RADIO and CHECKBOX to change the Deafult State of these Input Methods.
  3. DISABLED : Denies Access to an Input Method.
  4. MAXLENGTH : Indicates the Number of Characters one can enter into a Text Input Method. Used with TEXT and PASSWORD Input Methods. It doesn't effect the size of the Input Field displayed on the Screen.
  5. NAME : Gives a Name to the value you pass to the Form Processor. This eshtablishes a Name-Value pair for the Form Processor.
  6. READONLY : Specifies that the Visitor can't Change the contents of the Text Input Field.
  7. SIZE : Specifies the Width of the Input Field (in characters) Used with TEXT and PASSWORD Input Methods.
  8. SRC : Specifies the Relative or Absolute URL of the file to be loaded in the Floating Frame.
  9. TYPE : Specifies the Type of the Input Method.
    1. TEXT : Produces a simple One Line Text Input Field which is useful for obtaining simple data. For collecting Multiple lines of Text use TEXTAREA
    2. PASSWORD : Produces a simple One Line Text Input Field similar to TEXT but is used to get Password. User can't see the actual Password rather they see only *
    3. RADIO : Produces a Radio Button (in Groups of 2 or more) . It is used when the user is supposed to choose one out of many values SELECT can also be used for this purpose.
    4. CHECKBOX : Produces a Check Box (in Groups of 2 or more) . It is used when the user is supposed to choose one or more out of many values SELECT can also be used for this purpose.
    5. SUBMIT : Produces a Button which when clicked Submits all the Values to the Form Processor . BUTTON creates a Generic Button which can interact with
      Scripts.
    6. RESET : Produces a Button which when clicked sets all the Input Methods to their default or empty settings . BUTTON creates a Generic Button which can interact with Scripts.
    7. IMAGE : Replaces a Submit Button with an Image . Its behavior is identical to that of Submit except that the x, y co-ordinates of the mouse position over the image when selected are also sent to the Form Processor. BUTTON creates a Generic Button which can interact with Scripts.
    8. FILE : Takes a File as an Input . It is used to Upload Files.
    9. HIDDEN : Creates a Hidden Input Method .
  10. USEMAP : Indicates the URL of the Client Side Image Map to be used with the Form.
  11. VALUE : Sets the default value Input Method .

Syntax

<FORM>
<INPUT [ACCEPT="MIME Type"] [CHECKED] [DISABLED] [MAXLENGTH="n"] [NAME="NAME"] [READONLY] [SIZE="n"] [SRC="URL"] [TYPE="TYPE"] [USEMAP="URL"] [VALUE="VALUE"]>
.....
</FORM>

Example

<FORM NAME="FEG1" >
Name :
<INPUT TYPE="TEXT" SIZE="25">
Password :
<INPUT TYPE="PASSWORD" SIZE="25">
<INPUT TYPE="BUTTON" NAME="BUTTON1" VALUE="SUBMIT">
<INPUT TYPE="BUTTON" NAME="BUTTON1" VALUE="RESET">
.....
</FORM>
