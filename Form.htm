FORM

This is to be used in the BODY Tag. This Tag sets up a Container for a Form. This Tag includes many Tags such as BUTTON , INPUT , SELECT , TEXTAREA , etc.
This is a Container Tag and thus needs to be closed.

Attributes

  1. ACCEPT-CHARSET : Specifies the Character encoding for the Input Data that the Form Processor may Accept.
  2. ACCEPT : Specifies a list of MIME Types separated by Commas that the Form Processor will Handle carefully.
  3. ACTION : Specifies the Absolute or Relative Loaction of the Form Processing Application.
  4. ENCTYPE : Specifies the MIME Type used to Submit (POST) the Form to the Server. Default is application/x-www-form-urlencoded. Use multipart/form-data when the returned Document contains Files.
  5. METHOD : Specifies how the Form Data is to be Transmitted to the Form Processor.
    1. GET : The Form Data is given to the Form Processor in the form of an Environment Variable.
    2. POST : The Form Data is given to the Form Processor in the form of a Standard Input to a Program.
  6. NAME : Gives a Name to the value you pass to the Form Processor.
  7. TARGET : Identifies in which previously Named Frame the Output from the Form Processor should Appear .

Syntax

<FORM [ACCEPT-CHARSET="ACCEPT-CHARSET"] [ACCEPT="ACCEPT"] [ACTION="URL"] [ENCTYPE="ENCTYPE"] [METHOD="{GET,POST}"] [NAME="NAME"] [TARGET="TARGET"]>
...Form Elements...
</FORM>

Example

<FORM NAME="FEG1" >
Name :
<INPUT TYPE="TEXT" SIZE="25">
Address :
<TEXTAREA NAME="FEGTAR" COLS="20" ROWS="5">
</TEXTAREA>
<BUTTON TYPE="SUBMIT" NAME="BUTTON1" VALUE= "SUBMIT"> Submit </BUTTON>
<BUTTON TYPE="RESET" NAME="BUTTON1" VALUE= "RESET"> Reset </BUTTON>
</FORM>
